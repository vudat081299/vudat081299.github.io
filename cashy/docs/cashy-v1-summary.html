§ 02 6 màn hình chính

Sáu màn hình chính.

01

Đăng nhập

Entry khi chưa có session. 3 options: Apple (Tier 2, primary per Guideline 4.8), Google, và form Email + Password. Link "Đăng ký" cho user chưa có account.

02

Đăng ký

Chỉ cho email + password flow. Apple/Google auto-create account khi sign in lần đầu. Fields: email, password, confirm password, displayName. Sau đó → PIN setup embedded → Tổng quan.

03

Tổng quan / Statistics

Dashboard chính. Balance, charts spending, top categories, trend. Filter linh hoạt theo period/type/category. Toggle Roll-up / Strict khi filter theo parent category.

04

Settings

Profile, theme, security (Face ID, change PIN), export CSV, logout, delete account, about & privacy policy.

05

Add / Edit Transaction

Type (income/expense) chọn trước (Tier 1 button), amount (Decimal), multi-select categories (hierarchy picker — expand parent thấy children), occurredAt, note.

06

Add / Edit Category

Name, type (income/expense), icon (SF Symbol/emoji), color, parent (optional — nil cho root). System categories không xóa được.

§ 03 Authentication & Security

Ba lớp bảo mật.

A

Providers

Sign in with Apple bắt buộc per Guideline 4.8. Email + Password local — không server, password lưu Keychain với .userPresence.

AuthenticationServices GoogleSignIn 7.x Keychain
P

PIN

4 hoặc 6 chữ số (user chọn). Hash SHA-256 + per-install salt, lưu Keychain. Gắn với account. Change yêu cầu verify old PIN.

CryptoKit SHA-256 Keychain
B

Face ID / Touch ID

Qua LocalAuthentication. Gắn với app, không gắn account. Device-level verify, fallback PIN nếu fail.

LocalAuthentication Per-app
Password recovery: v1 không hỗ trợ. User quên password → (a) dùng Apple/Google làm alternative, hoặc (b) delete + reinstall (mất data). Có thể reconsider ở v2+.

§ 05 Architecture v1

Modular layered + MVVM.

MVVM first cho mọi feature (per vision §3 code principle), trong khung Modular layered architecture với 3 layer Presentation / Domain / Data. Refactor lên TCA khi feature explosion — 21 UC + 6 màn hình chưa đủ phức tạp.

Dependencies point inward.

Domain ở trung tâm — pure Swift, không phụ thuộc framework.

System fallback category

"Pending" — Chưa phân loại.

Một system category đặc biệt (isSystem: true, name "Chưa phân loại", icon questionmark.circle, color gray) seed cho mọi user lúc account tạo. Không xóa được. Có hai vai trò:

Role 01 · Quick-entry default
Nhập nhanh, hoàn tất sau.

User mở app, tap "+", nhập amount + type, lưu nhanh (chưa chọn category) → transaction tự gán categories = [Pending]. Tối về nhà, user mở lại để gán category thật.

Role 02 · Orphan fallback
Invariant: transaction luôn có ≥ 1 category.

Khi user xóa category, SwiftData M:N tự remove khỏi transaction.categories. Nếu sau cleanup categories.isEmpty → repository tự thêm Pending. Transactions không bao giờ mồ côi.

Mixed-type categories: vì Transaction.type explicit và Category.type chỉ để classify UI, một transaction có thể gắn categories có type khác nhau. Stats dùng Transaction.type là source of truth. Open question (defer v2): có nên cho phép mix tự do hay restrict — revisit khi có data thực tế.

§ 08 Appendix · DO / DON'T cho v1

Khi code Cashy v1.

Do

Luôn áp dụng
  • Dùng SwiftUI + SwiftData + @Observable + Swift 6.2 strict concurrency
  • Tuân thủ Modular layered + MVVM: Presentation ↔ Domain ↔ Data, dependency point inward
  • Repository pattern qua protocol — không bao giờ gọi SwiftData/Keychain trực tiếp từ Presentation
  • Password → Keychain trực tiếp (HW-backed). PIN → SHA-256 + salt rồi lưu Keychain
  • Face ID qua LocalAuthentication, gắn với app
  • SwiftData operations qua @ModelActor cho concurrency safety
  • Money = Decimal, không Double
  • Transaction.type explicit (income/expense) — không derive từ Category.type
  • Quick-entry default → categories = [Pending] nếu user không chọn
  • Invariant: transaction luôn có ≥ 1 category; cleanup isEmpty → tự gán Pending
  • Cascade delete parent category → children (SwiftData @Relationship(deleteRule: .cascade))
  • Stats roll-up toggle trong UI — default ON khi filter parent category
  • Async/await thay vì closure

Don't

Không bao giờ
  • Không import SwiftData/Keychain trong Domain layer
  • Không dùng Combine, ObservableObject
  • Không dùng UIKit khi có thể dùng SwiftUI
  • Không thêm feature ngoài 21 UC mà chưa thảo luận
  • Không kết nối network mặc định
  • Không add analytics, ads, tracking
  • Không dùng external crypto lib
  • Không tự xử lý password recovery — v1 không support
  • Không cho xóa Pending (cũng như mọi isSystem category)
  • Không derive Transaction.type từ categories — luôn dùng explicit field
  • Không hardcode depth limit trong schema — soft limit ở UI nếu cần
