regular_expression Regex (the “Regex” mode)

          

            
ExampleMeaning

            
              
priceContains “price” (case-insensitive)

              
^userStarts with “user” — e.g. user, username

              
id$Ends with “id” — e.g. id, userId

              
^total$Exactly “total”, nothing more

              
name|emailContains “name” or “email”

              
a.b. = any one character — “a”, one char, “b” (e.g. axb)

              
a.*b“a”, then any characters (or none), then “b” — ab, a123b

              
a.+b“a”, then one or more characters, then “b”

              
a.{2,3}b“a”, then 2 to 3 characters, then “b”

              
\d{4}Four digits in a row — e.g. year 2026

              
-?\d+An integer, optionally negative

              
[aeiou]Any one of the characters in the brackets

              
[^0-9]Any character that is not a digit

              
colou?rThe char before ? is optional — “color”/“colour”

              
(cat|dog)s?A group + optional “s” — cat, cats, dog, dogs

              
\.vn\bContains “.vn” (\. = a literal dot)

            
          

        

        

          
account_tree Path (the “Path” mode)

          

            
ExampleMeaning

            
              
userThe user field at the root (with its whole subtree)

              
user.coPrefix match: user.code, user.cowork…

              
user.addr.cityDrill down level by level with dots

              
items.0 · items[0]First element of the items array

              
items.*.sku* = any one level — the sku of every element

              
**.price** = any depth — every price anywhere

              
orders.*.totaltotal in each order of orders

            
          

          

            Tip: it filters as you type — the last segment matches by prefix, so partial input still returns results.
